# Support

Thank you for using Cursor-Claude Compat! This document outlines how to get help and what kind of support you can expect.

## Getting Help

### Documentation

Before reaching out, please check:
- [README.md](README.md) - Overview and quick start guide
- [docs/](docs/) - Detailed documentation
- [CONTRIBUTING.md](CONTRIBUTING.md) - Contribution guidelines
- [Existing Issues](https://github.com/tochitatebuilding/cursor-claude-compat/issues) - Your question may have been answered

### Issue Tracker

For bug reports, feature requests, and questions:

1. **Search existing issues** to see if your question has been asked
2. **Use the appropriate template**:
   - [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md)
   - [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md)
3. **Provide clear information**:
   - What you're trying to do
   - What happened vs. what you expected
   - Steps to reproduce (for bugs)
   - Your environment (OS, shell version, etc.)

### Priority Levels

- **Critical Bugs**: Issues that prevent the tool from working at all
- **Feature Requests**: Enhancements and new functionality
- **Questions**: Usage questions and clarifications
- **Documentation**: Improvements to documentation

## Support Scope

### What We Support

✅ **Supported**:
- Bug fixes and issues with the codebase
- Feature requests and enhancements
- Documentation improvements
- Usage questions and clarifications
- Security vulnerabilities (see [SECURITY.md](SECURITY.md))

### What We Don't Support

❌ **Not Supported**:
- Custom integrations or modifications
- Commercial support or consulting services
- Guaranteed response times or SLAs
- Support for modified versions of the software
- Issues with third-party tools or dependencies

## Response Times

We aim to respond to issues within a reasonable timeframe, but please note:

- **Best Effort**: This is an open-source project maintained by volunteers
- **No SLA**: We don't guarantee specific response times
- **Priority**: Critical bugs and security issues receive priority
- **Community Help**: Other users may respond faster than maintainers

## Community Support

The best way to get help is often from the community:

- **Search Issues**: Many questions have been answered before
- **Help Others**: Answer questions if you can
- **Contribute**: Submit PRs for documentation improvements

## Commercial Support

## AI Usage Transparency

**Note**: This project uses AI assistance for certain automated tasks:

- **Issue responses**: Some issue responses may be generated or assisted by AI
- **Documentation**: Documentation may be enhanced with AI assistance

All AI-generated content is reviewed by maintainers. We believe in transparency about AI usage in open-source projects.

## Reporting Problems

If you encounter a problem:

1. **Check the documentation** first
2. **Search existing issues** for similar problems
3. **Create a new issue** with clear details
4. **Be patient** - maintainers are volunteers

## Contributing Back

The best way to support the project is to contribute:

- **Report bugs** with clear reproduction steps
- **Suggest features** with use cases
- **Submit PRs** for fixes and improvements
- **Improve documentation** based on your experience
- **Help others** in issues and discussions

Thank you for using Cursor-Claude Compat!
