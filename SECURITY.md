# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public issue. Instead, please report it privately using one of the following methods:

### Preferred Method: Security Email

Email: **system@tochitatebuilding.co.jp**

Please include:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (if available)

### Response Timeline

- **Acknowledgment**: Within 72 hours
- **Initial Assessment**: Within 7 business days
- **Critical Vulnerabilities**: Priority handling with a target fix within 14 days

### What to Expect

1. **Confirmation**: We'll acknowledge receipt of your report within 72 hours
2. **Assessment**: We'll assess the vulnerability and determine its severity
3. **Fix**: We'll work on a fix and keep you updated on progress
4. **Disclosure**: After a fix is released, we'll coordinate disclosure timing with you

### Security Response Process

1. **Report Received**: We acknowledge receipt
2. **Triage**: We assess severity and impact
3. **Fix Development**: We develop and test a fix
4. **Release**: We release the fix in a security update
5. **Disclosure**: We coordinate public disclosure (if appropriate)

## Scope

### In Scope

- Security vulnerabilities in the codebase
- Authentication/authorization issues
- Data exposure or leakage
- Injection vulnerabilities
- Insecure dependencies

### Out of Scope

- Denial of service attacks
- Social engineering attacks
- Physical security issues
- Issues requiring physical access to systems
- Issues in third-party dependencies (please report to the maintainer of that dependency)

## AI Usage Transparency

**Important**: This project uses AI assistance for certain automated tasks:

- **Issue responses**: Some issue responses may be generated or assisted by AI
- **Security analysis**: Automated security checks may use AI assistance
- **Vulnerability assessment**: Initial assessments may be assisted by AI

All security-related decisions and fixes are reviewed by human maintainers. We believe in transparency about AI usage in security processes.

## Security Best Practices

When using this project:

- Keep your system and dependencies up to date
- Review configuration files before use
- Use secure methods for storing sensitive information
- Follow the principle of least privilege
- Regularly review access permissions

## Recognition

We appreciate responsible disclosure. With your permission, we'll acknowledge security researchers who help improve the security of this project.

## Questions

If you have questions about this security policy, please contact us at system@tochitatebuilding.co.jp.

Thank you for helping keep Cursor-Claude Compat secure!
